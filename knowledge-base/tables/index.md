# Tables

> Generated by `rake kb:index` — do not edit by hand; change the source file's `description` and regenerate.

* [allocation_categories](allocation_categories.md) - Household-scoped grouping buckets for allocations (food, household, transport, …). Carries two orthogonal axes: budget_role (newer, drives NLQ analysis sectioning and seeds class) and the seeded allocation_class default.
* [allocations](allocations.md) - The core money-out unit of EveryCent. One budgeted outflow line per budget; where plan meets actual via transactions.allocation_id. Carries a four-way class, a category, and behavior flags (several dead/incomplete).
* [bank_accounts](bank_accounts.md) - Where money physically lives, and the execution site for budget close, brought-forward, manual adjustment, and transfers. Two orthogonal axes: account_category (asset/liability/current) and account_type (normal/credit_card/sink_fund) which gates feature concerns.
* [budgets](budgets.md) - The monthly period container at the heart of the zero-based engine. Incomes and allocations attach to a budget; transactions link only indirectly.
* [chat_settings](chat_settings.md) - One config row per household for the NLQ chat feature — chat_enabled, the selected llm_model_id, max_tool_iterations, and an extras jsonb. Read at the start of every chat turn.
* [households](households.md) - The tenant root and isolation boundary. Almost every live table is scoped to a household via a direct household_id foreign key.
* [incomes](incomes.md) - Plan-only inflow lines, one set per budget. Declares expected income that allocations budget against. Not linked to actual deposit transactions.
* [institutions](institutions.md) - Household-scoped lookup of financial institutions (e.g. a retail bank). Informational; bank accounts optionally reference one. Nothing keys off it functionally.
* [llm_models](llm_models.md) - Registry of LLM models a household can pick for the NLQ chat. Holds the endpoint URL and five cost-rate columns (cents per million tokens). CRUD-managed; one row per [household, provider, name].
* [llm_usage_records](llm_usage_records.md) - One row per LLM call in a chat turn. Captures tokens, a snapshot of the model's cost rates, computed total_cost, conversation/turn UUIDs, and tool telemetry. The current point of the feature: logging to tune chat configuration.
* [sessions](sessions.md) - The live auth token store. One row per active login; holds token_digest (never the raw token), a sliding 7-day expiry, and audit fields. Replaced devise_token_auth's in-user token store.
* [settings](settings.md) - A per-household singleton config row, accessed as a class-level key-value store. Holds the primary budget account, household composition (names + family_type), and the special-events category default.
* [sink_fund_allocations](sink_fund_allocations.md) - Named money-storing envelopes inside a sink-fund bank account. The live balance is transaction-derived (current_balance); the amount/target field is near-vestigial (display-only, slated for removal).
* [special_events](special_events.md) - A discrete non-recurring spending occasion (vacation, birthday, Christmas) that groups allocations across multiple budgets. Stored budget_amount/actual_amount are a frontend-maintained cache; the truth is the allocation rollup.
* [transactions](transactions.md) - One bank transaction — actual money movement synced from real bank data. Where plan (allocations) meets reality. Budget membership is by date range, not a stored FK. allocation_id presence marks budget membership.
* [users](users.md) - Login identity. belongs_to one household (N users per household — both partners use the same one). Email is the unique Google join key. The bulk of the columns are vestigial devise/devise_token_auth fields left by the additive auth rebuild.

## Dead / partial

* [payees](payees.md) _(dead)_ - Abandoned v1 payee list (Trinidad-era). Each payee had a default allocation so a transaction could be tagged by resolving its description to a payee. Never built out; empty in prod, no model, no household_id. Replaced by auto-allocation.
* [recurring_allocations](recurring_allocations.md) _(dead)_ - Abandoned v1 templates for auto-populating a new budget's allocations on create. Never truly implemented; replaced by copy budget. Empty in prod, no model, no household_id.
* [recurring_incomes](recurring_incomes.md) _(dead)_ - Abandoned v1 templates for auto-populating a new budget's incomes on create. The income half of recurring allocations; never truly implemented, replaced by copy budget. Empty in prod, no model, no household_id.
